## Key Architecture Decision
All 4 tabs now use PostgREST directly instead of edge functions for list views. This enables offset pagination (numbered pages), `Prefer: count=exact` for totals, and uniform filter/search handling. Edge functions remain deployed for the old prod site.

PostgREST returns HTTP 416 when offset exceeds total — Content-Range header is still valid. API layer handles this as empty results.
